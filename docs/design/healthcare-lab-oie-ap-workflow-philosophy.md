# Container Signal

Container Signal frames integration as a controlled clinical circuit. The container boundary is not background decoration; it is the governing architectural form, holding internal services in one deliberate enclosure while the external AP remains visibly outside. The image should feel meticulously crafted, with the operational boundary communicated through scale, contrast, and disciplined spacing.

The workflow depends on direction. Arrows carry the essential meaning, so each path must be treated as a precise signal lane with sparse, exact labels. Message type and port are the typography of the system: ORM and ORU should be immediately legible without turning the piece into a table. This requires painstaking attention to placement and line rhythm.

Color is functional and restrained. ORM and ORU use distinct signal colors, while the Docker container boundary uses a cooler technical tone. The composition should feel calm enough for a presentation, but exact enough for engineers to trust it. Every label, curve, and node should show master-level execution.

The final artifact should look like it was refined through many small passes: no collisions, no accidental hierarchy, no wasted ornament. The three major blocks must read instantly, and the message path must remain clear from across a room.
