# Healthcare Lab v1.0.1

## Fixed

- Fixed Docker-based lab-app connections to dcm4chee by using the internal
  `dcm4chee` service name for DIMSE, HL7, QIDO-RS, WADO-RS, STOW-RS, and MWL.
- Added dcm4chee archive-storage initialization to prevent C-STORE processing
  failures caused by `/storage/fs1` permissions.
- Aligned the default OIE host ports with ports 8080 and 8443.
- Fixed the GDT patient workflow list to display the canonical patient MRN
  instead of the internal database record ID.
- Updated English and Traditional Chinese deployment documentation.

## Upgrade notes

Existing `.env` files are not replaced automatically. Existing Docker users
should change dcm4chee internal endpoints from `127.0.0.1` to `dcm4chee`, or
reconcile their configuration with the new `.env.example`.

The default OIE host publications are now 8080 and 8443. Users who need the
previous 18080 and 10443 ports can explicitly set `OIE_HTTP_PORT=18080` and
`OIE_HTTPS_PORT=10443`.

The default lab-app image is now
`ghcr.io/tzu-huang/healthcare-lab:1.0.1`.
