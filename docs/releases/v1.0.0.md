# Healthcare Lab v1.0.0

Healthcare Lab's first deployable release provides a Docker Compose-based
environment for experimenting with healthcare interoperability workflows.

## Highlights

- Supports HL7 v2, FHIR/Medplum, GDT 2.1, and DICOM/dcm4chee workflows.
- Provides patient-centered dashboards, order/result views, and integration
  service diagnostics.
- Provides managed OIE Channels, the HLAB ORU listener, and runtime diagnostics.
- Provides local persistence, synchronization retry, result reconciliation,
  and audit information.
- Packages `lab-app` as a self-contained production Gunicorn image.
- Supports Docker Compose deployment on a trusted local machine or internal lab
  network.

## Container distribution

- Registry: `ghcr.io/tzu-huang/healthcare-lab`
- Platform: `linux/amd64`
- Stable tags: `1.0.0`, `1.0`, `1`, `latest`, and `sha-<commit>`
- Main branch tags: `edge` and `sha-<commit>`

The stable release workflow first verifies that the GHCR package is anonymously
readable and prevents an immutable version tag from being overwritten by a
different commit.

## Important limitations

- Only trusted local machines and internal lab networks are supported.
- Public-Internet, regulated production, ARM, and multi-replica deployments are
  not supported by this release.
- The Docker socket mount grants host-level control. Operators must understand
  and accept this security boundary.

## Upgrade and persistence

SQLite instance data and GDT bridge data live in external volumes and survive
application-container replacement. See `docs/container-release.md` for upgrade,
backup, and rollback procedures.
