# Healthcare Lab v1.0.0 Release Checklist

This checklist prepares the first public Docker release. Completing
implementation does not create the Git tag, publish the GitHub Release, or move
GHCR stable tags; those actions happen only after the dev verification and
review gates approve one exact commit.

## Release artifacts

- [ ] The approved commit contains `Dockerfile`, `.dockerignore`,
  `deploy/docker-compose.yml`, `.env.example`, and current operator docs.
- [ ] OpenSpec verification and closure review approve the exact release commit.
- [ ] The full Python suite, container contract tests, Compose validation, image
  build, clean-container smoke check, listener check, and persistence check pass.
- [ ] Image inspection confirms no `.env`, credentials, Git metadata, local
  instance data, tests, caches, or other excluded content is present.
- [ ] `linux/amd64` is the only advertised platform and the integrated
  third-party digest matrix matches `docs/container-release.md`.

## Publication

- [ ] Create Git tag `v1.0.0` on the approved commit and publish a stable,
  non-prerelease GitHub Release from that tag.
- [ ] Confirm the release workflow publishes public GHCR tags `1.0.0`, `1.0`,
  `1`, `latest`, and `sha-<commit>`.
- [ ] Confirm `1.0.0` and the SHA tag resolve to the approved source revision and
  record the resulting image digest in the final release notes.
- [ ] Confirm an unauthenticated `docker pull
  ghcr.io/tzu-huang/healthcare-lab:1.0.0` succeeds.

## Post-publication smoke

- [ ] On a clean `linux/amd64` Docker environment, use only the downloaded
  release files, local `.env`, and published images to start Compose.
- [ ] Confirm the root page returns HTTP 200, static assets render, SQLite is on
  the named volume, and the OIE result listener reports `running` on port 6665.
- [ ] Confirm a container replacement retains instance and GDT bridge data.
- [ ] Confirm a later `main` build updates only `edge`/SHA and does not change
  the `1.0.0` digest.
